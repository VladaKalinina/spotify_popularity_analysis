# Какие аудиохарактеристики связаны с популярностью треков Spotify?

Учебный проект по дисциплине **«Анализ данных на Python»**.

**Команда:** Калинина Влада, Худякова Светлана

## Цель проекта

Исследовать, какие аудиохарактеристики связаны с популярностью треков на Spotify, проверить статистические гипотезы и оценить, насколько хорошо популярность можно прогнозировать с помощью методов машинного обучения.

Проект изучает статистические связи между показателями. Полученные результаты не доказывают причинного влияния отдельных характеристик на популярность трека.

## Исследовательские вопросы и гипотезы

В проекте рассматриваются следующие вопросы:

1. Отличаются ли по популярности треки с пометкой `explicit`?
2. Связана ли танцевальность трека с его популярностью?
3. Популярнее ли треки в мажоре, чем треки в миноре?
4. Можно ли оценить популярность трека по его аудиохарактеристикам?
5. Улучшают ли данные Last.fm качество модели?

Проверяемые статистические гипотезы:

- **Гипотеза 1.** Explicit-треки в среднем популярнее треков без такой пометки.
- **Гипотеза 2.** Танцевальность положительно связана с популярностью трека.
- **Гипотеза 3.** Треки в мажоре в среднем популярнее треков в миноре.

## Источники данных

### Основной датасет

Основной источник — [Spotify Tracks Dataset на Kaggle](https://www.kaggle.com/datasets/maharshipandya/-spotify-tracks-dataset).

Исходный набор содержит около **114 000 треков** и включает:

- название трека и исполнителя;
- музыкальный жанр;
- показатель популярности;
- длительность;
- пометку `explicit`;
- танцевальность, энергичность, громкость, акустичность;
- инструментальность, живость, валентность, темп и другие признаки.

### Обогащение данных через Last.fm API

Для выполнения сложного сбора данных исходный датасет был дополнен информацией из **Last.fm API**.

Отдельный скрипт `src/enrich_lastfm.py`:

- формирует выборку из **3000 популярных артистов**;
- обеспечивает представительство разных музыкальных жанров;
- отправляет запросы к Last.fm API;
- получает число слушателей, число прослушиваний и пользовательские теги;
- обрабатывает ошибки запросов и повторные попытки;
- сохраняет промежуточные результаты;
- позволяет продолжить загрузку после прерванного запуска.

После сбора были получены данные для **2998 артистов**. Реальные Last.fm-признаки доступны для **52 071 трека** исходной таблицы.

Таким образом, в проекте использован сложный готовый датасет, который был очищен, преобразован и обогащён через сторонний API на крупной выборке с жанровым покрытием.

1. Загрузка основного датасета Spotify.
2. Формирование выборки артистов и сбор данных через Last.fm API.
3. Объединение источников данных.
4. Проверка типов, пропусков, дубликатов и аномальных значений.
5. Создание новых расчётных признаков.
6. Разведочный анализ и визуализация.
7. Проверка трёх статистических гипотез.
8. Обучение и сравнение моделей машинного обучения.
9. Оценка дополнительной пользы Last.fm-признаков.

## Предобработка данных

В ходе очистки и подготовки данных:

- удалены повторяющиеся записи по `track_id`;
- проверены типы данных и допустимые диапазоны значений;
- пропуски аудиопризнаков заполнены медианой внутри жанра и затем общей медианой;
- проанализированы крайние значения и потенциальные выбросы;
- данные Last.fm не заменялись искусственными значениями при отсутствии ответа API;
- созданы новые признаки, включая длительность в минутах, категорию популярности, бинарные признаки и логарифмы показателей Last.fm.

После очистки финальная аналитическая таблица содержит **89 741 трек** и **44 признака**.

## Методы анализа

В проекте использовались:

- описательная статистика;
- группировки и агрегирование;
- корреляции Пирсона и Спирмена;
- Welch t-test;
- размер эффекта Cohen’s d;
- визуальный разведочный анализ;
- линейная регрессия;
- дерево решений;
- Random Forest;
- baseline-модель `DummyRegressor`;
- кросс-валидация;
- метрики MAE, RMSE и R².

## Основные результаты

- Explicit-треки в среднем популярнее на **4,03 пункта**, однако размер эффекта небольшой: **Cohen’s d = 0,196**.
- Связь танцевальности с популярностью положительная, но очень слабая: **Pearson r = 0,0643**, **Spearman ρ = 0,0553**.
- Гипотеза о том, что мажорные треки популярнее минорных, не подтвердилась. Разница составила **−0,69 пункта**, **Cohen’s d = −0,034**.
- Лучшей моделью оказался **Random Forest**: **test R² = 0,241**, **test RMSE = 17,81**.
- Добавление Last.fm-признаков повысило R² на **0,006**, то есть дало небольшой дополнительный сигнал.

Главный вывод: аудиохарактеристики связаны с популярностью, но сами по себе объясняют её лишь частично. На результат также влияют известность исполнителя, продвижение, попадание в плейлисты, дата релиза и музыкальные тренды.

## Запуск проекта

### 1. Создание виртуального окружения

```bash
python -m venv .venv
source .venv/bin/activate
```

Для Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Добавление основного датасета

Скачайте `dataset.csv` с Kaggle и поместите файл по адресу:

```text
data/raw/dataset.csv
```

### 4. Настройка Last.fm API

Скопируйте `.env.example` в `.env` и добавьте API-ключ:

```text
LASTFM_API_KEY=ваш_ключ
```

### 5. Запуск сбора данных

```bash
python src/enrich_lastfm.py
```

### 6. Запуск анализа

```bash
jupyter lab
```

Откройте файл:

```text
notebooks/spotify_popularity_analysis.ipynb
```

и выполните все ячейки последовательно.

## Ограничения исследования

- показатель Spotify `popularity` является платформенной метрикой и может изменяться со временем;
- аудитории Spotify и Last.fm различаются;
- данные не содержат всех факторов маркетинга и продвижения;
- несколько треков одного артиста не являются полностью независимыми наблюдениями;
- обнаруженные статистические связи не доказывают причинность;
- результаты нельзя считать гарантированным прогнозом успеха будущего релиза.
